¿Qué es el proyecto?
El proyecto consiste en crear una aplicación web sencilla para la gestión de tareas, realizada en equipo siguiendo la metodología Scrum. La aplicación permite crear y visualizar tareas, aplicando reglas de negocio básicas.

Requisitos:
Para ejecutar el proyecto en local es necesario tener instalado:
GitHub Desktop para trabajo colaborativo
Anaconda
Visual

¿Cómo arrancarlo en local?
1. Clonar el repositorio.
2. Acceder a la carpeta del proyecto.
3. Crear y activar un entorno virtual.
4. Instalar dependencia.
5. Arrancar el backend.
6. Abrir el frontend.

Estructura de proyecto:
1. backend/
  - Contiene el código del servidor y la lógica de la aplicación.
2. frontend/
  - Incluye la parte visual de la aplicación.
3. docs/
  - Carpeta destinada a documentación adicional del proyecto (decisiones, notas, capturas, etc.).
4. .gitignore
  - Define los archivos y carpetas que no deben subirse al repositorio (entornos virtuales, cachés, etc.).
5. README.md
  - Documento principal del proyecto con la descripción, instrucciones y documentación básica.
